## Задача проекта

Оценить эффективность трёх команд разметки (Альфа, Бета, Гамма) за май 2026 года по ключевым метрикам:
- объём выполненной работы (паблики)
- скорость разметки (пабликов в час)
- качество (количество ошибок)
- дисциплина (нарушения)

Цель — выявить лидеров, зоны роста и дать руководителю отдела конкретные рекомендации для повышения эффективности.

## Данные

- **Источник:** анонимизированная выгрузка из системы учёта задач (Excel)
- **Период:** май 2026
- **Объекты:** 15 сотрудников, 3 команды
- **Метрики:** количество размещённых пабликов, скорость, количество ошибок, количество нарушений, тип карточки (зелёная — высокий уровень, белая — средний)

## Инструменты

- **DataLens** — создание интерактивного дашборда
- **Pandas (Python)** / **Excel Power Query** — очистка и агрегация данных (код в `/notebooks`)
- **Git** — контроль версий

## Ключевые выводы

### Общие показатели
- Всего размещено пабликов: **29 160**
- Максимум от одного сотрудника: **2 150** пабликов
- Максимальная скорость: **13,44** паблика/час
- Всего нарушений: **8**

### Команды
| Команда | Паблики | Ошибки | % ошибок | Нарушения |
|---------|---------|--------|----------|------------|
| Гамма   | 9 910   | 77     | 0,78%    | 3          |
| Альфа   | 9 530   | 77     | 0,81%    | 2          |
| Бета    | 9 720   | 95     | 0,98%    | 3          |

**Лидеры:**
- По объёму: Гамма (9 910)
- По точности: Гамма и Альфа (0,78–0,81% ошибок)
- По дисциплине: Альфа (2 нарушения)

**Зоны внимания:**
- Команда Бета показывает самый высокий процент ошибок (0,98% при среднем 0,86%)
- Белых карточек больше, чем зелёных (9 против 6) — есть потенциал для роста
- Наблюдается общая корреляция: с ростом скорости количество ошибок в среднем увеличивается, но есть опытные сотрудники с высокой скоростью и низким числом ошибок

## Рекомендации

1. **Тренинг для команды Бета** – снизить количество ошибок до уровня отдела (целевой % ошибок < 0,85%)
2. **Командный план по зелёным карточкам** – минимум 3 зелёные карточки на команду в следующем месяце
3. **Добавить динамику** – в следующих версиях дашборда сравнить с предыдущим месяцем для отслеживания прогресса
4. **Автоматическая проверка** – для сотрудников со скоростью выше 13 пабликов/час ввести дополнительную выборочную проверку, чтобы снизить число ошибок

## Как использовать дашборд

Дашборд интерактивный. Вы можете:
- наводить на графики для просмотра значений
- (если добавлены фильтры) выбирать конкретную команду или сотрудника
- анализировать связь скорости и ошибок на диаграмме рассеяния
